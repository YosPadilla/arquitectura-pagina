1.2 Análisis de los componentes

Esta sección profundiza en los elementos esenciales que componen una arquitectura de cómputo moderna, desde la unidad central de procesamiento hasta los buses de comunicación e interrupciones del sistema.

1.2.1 CPU

1.2.1.1 Arquitecturas

Existen dos tipos predominantes de arquitectura de CPU: RISC (Reduced Instruction Set Computer) y CISC (Complex Instruction Set Computer). RISC se caracteriza por instrucciones simples y rápidas que requieren múltiples pasos para tareas complejas. CISC utiliza instrucciones más elaboradas, capaces de ejecutar tareas complejas en una sola instrucción. RISC es eficiente en dispositivos móviles, mientras que CISC se mantiene vigente en arquitecturas x86.

1.2.1.2 Tipos

Las CPUs pueden clasificarse en:

  • Mononúcleo: Procesador con una sola unidad de ejecución.
  • Multinúcleo: Contiene múltiples núcleos que permiten ejecutar varias instrucciones simultáneamente.
  • CPU embebidas: Integradas en dispositivos como microcontroladores, electrodomésticos, automóviles, etc.

1.2.1.3 Características

Las principales características que definen el desempeño de una CPU son:

  • Velocidad de reloj: Medida en GHz, indica cuántos ciclos de instrucciones puede realizar por segundo.
  • Memoria caché: Memoria interna ultrarrápida dividida en niveles (L1, L2, L3).
  • Longitud de palabra: Cantidad de bits que puede procesar simultáneamente.
  • Set de instrucciones: Conjunto de comandos que puede entender y ejecutar.

1.2.1.4 Funcionamiento (ALU, Unidad de Control, Registros y Buses Internos)

La CPU está compuesta por:

  • ALU (Unidad Aritmético-Lógica): Ejecuta operaciones matemáticas y lógicas.
  • Unidad de control: Coordina el flujo de datos e instrucciones dentro de la CPU.
  • Registros: Memoria interna de acceso ultrarrápido que almacena temporalmente datos e instrucciones en ejecución.
  • Buses internos: Canales de comunicación entre las unidades dentro de la CPU.

1.2.2 Memoria

1.2.2.1 Conceptos básicos del manejo de la memoria

La memoria es un componente que almacena datos e instrucciones. Su manejo eficiente determina el rendimiento del sistema. Las operaciones básicas incluyen lectura (read), escritura (write), direccionamiento y temporización de acceso.

1.2.2.2 Memoria principal semiconductora

Se refiere a la memoria RAM y ROM construidas con circuitos semiconductores. La RAM (volátil) permite lectura y escritura rápida, mientras que la ROM (no volátil) almacena datos permanentes como el firmware del sistema.

1.2.2.3 Memoria caché

Es una memoria intermedia entre la CPU y la RAM que almacena datos e instrucciones usados frecuentemente. Existen niveles jerárquicos: L1 (más rápida, interna al núcleo), L2 (externa al núcleo, pero dentro del chip), y L3 (compartida entre núcleos). Mejora el rendimiento reduciendo el tiempo de acceso a la memoria principal.

1.2.3 Manejo de la entrada/salida

1.2.3.1 Módulos de entrada/salida

Son interfaces que permiten conectar dispositivos periféricos al sistema, como teclados, pantallas, impresoras o sensores. Estos módulos gestionan señales eléctricas, protocolos de comunicación y sincronización de datos.

1.2.3.2 Entrada/salida programada

En este modo, el procesador ejecuta instrucciones específicas para controlar los dispositivos. Es simple, pero ineficiente, ya que el procesador debe esperar mientras el dispositivo responde.

1.2.3.3 Entrada/salida mediante interrupciones

El periférico interrumpe al procesador cuando necesita atención, liberando al CPU de la espera activa. El sistema responde con una rutina de atención a interrupciones (ISR), lo que mejora la eficiencia general.

1.2.3.4 Acceso directo a memoria (DMA)

El controlador DMA transfiere datos entre el periférico y la memoria sin intervención del CPU. Se utiliza en tareas de alto volumen de datos como transferencias de disco o audio.

1.2.3.5 Canales y procesadores de entrada/salida

En sistemas de alto rendimiento, se emplean canales de E/S especializados que procesan tareas de entrada/salida independientemente del CPU principal, permitiendo trabajar en paralelo.

1.2.4 Buses

1.2.4.1 Tipos de buses

Los buses se clasifican en:

  • Bus de datos: Transporta los datos procesados.
  • Bus de direcciones: Indica la dirección de memoria o registro a acceder.
  • Bus de control: Transporta señales de sincronización y control.

1.2.4.2 Estructura de los buses

Los buses pueden ser paralelos (varios bits transmitidos simultáneamente) o seriales (bits enviados secuencialmente). Un bus típico se compone de líneas para cada tipo de señal (datos, control, dirección) y opera bajo protocolos definidos por el chipset.

1.2.4.3 Jerarquías de buses

En sistemas complejos, existen jerarquías como el bus del sistema (system bus), bus local (local bus) y buses periféricos (USB, PCI, SATA), que se comunican entre sí mediante puentes lógicos (bridges).

1.2.5 Interrupciones

Las interrupciones son señales que provocan una alteración en el flujo normal del programa para atender eventos urgentes. Pueden ser:

  • Interrupciones internas (traps): Generadas por errores del programa (división por cero, overflow).
  • Interrupciones externas: Provienen de dispositivos periféricos (teclado, disco, red).

Cada interrupción está asociada a un vector que redirige la ejecución a una rutina especial llamada ISR (Interrupt Service Routine). El manejo eficiente de interrupciones permite multitarea, respuesta rápida y optimización del rendimiento.
